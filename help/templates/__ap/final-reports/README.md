# final-reports

## Objetivo
- Consolidar, de forma didática e auditável, tudo o que foi executado em um AP.

## Arquitetura recomendada
- `report-final.md` (mestre obrigatório)
- `report-final-execucao.md`
- `report-final-arquitetura-e-contratos.md`
- `report-final-riscos-e-pendencias.md`
- `final-reports-checklist.md`

## Regras importantes
- Pode existir mais de um report final complementar.
- O `report-final.md` mestre é obrigatório e deve indexar os anexos.
- Sem pacote completo, o AP não deve migrar de PTBE para PE.
