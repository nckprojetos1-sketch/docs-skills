# __an template

## Objetivo
- Modelo inicial para o caminho AN (Analyze) no método DOCS.

## Status
- Coming Soon!

## Proposito do AN
- Analisar repositórios existentes de forma estruturada e auditável.
- Cobrir diagnóstico de:
  - código-fonte
  - banco de dados
  - endpoints e contratos de API

## Saidas esperadas
- Inventário técnico do estado atual.
- Mapa de riscos, lacunas e inconsistências.
- Base documental para decisão de execução futura em AP ou SDDR.

## Fronteira do caminho
- AN é uma trilha de análise e diagnóstico.
- AN não substitui AP/SDDR para execução de mudanças no estado atual do método.
